pdfalo

An exercise in style: the same PDF syntax the other parsers read, this time through GNU Bison’s C++ lalr1.cc skeleton — a deliberately small, non-validating LALR parser producing a move-only AST. The grammar stays honest about one useful distinction: an indirect object’s body is an ordered sequence of syntactic objects, so a dictionary followed by a stream is two adjacent objects, not one thing pretending to be another.

It parses; it does not validate and it does not model. Alongside ypdf’s recursive descent it is scored against xpdf’s object dumps over a shared corpus — three parsers, three paradigms, one oracle.
